# Motion Graphics Concept Catalog

A catalog of video concept archetypes for Remotion video creation. The remotion skill draws from this catalog dynamically based on the creative brief. Concepts can be combined (hybrid). New concepts can be added without restructuring the skill.

---

## How This Catalog Is Used

1. The creative brief (Step 1) defines: audience, content type, channels, core message, CTA
2. In Step 2, the agent scores each concept against the brief using the Selection Criteria below
3. The top 3-5 concepts are presented as the variant options
4. The user selects one (or a hybrid of two)

---

## Selection Criteria

For each concept, evaluate against the brief:

| Factor | Weight | Question |
|---|---|---|
| Content-type fit | High | Does the concept's "Best For" match the content type? |
| Message clarity | High | Can the core message land in 15-30 seconds with this structure? |
| Cold audience stopping power | High | Can a zero-context viewer understand the value in the first 3 seconds? |
| Channel fit | Medium | Does the concept work at the target dimensions and duration? |
| Available assets | Medium | Does the brief provide the data/visuals this concept needs? |
| CTA delivery | Medium | Does the concept naturally lead to the intended CTA? |
| Differentiation | Low | Has this concept been used recently for this brand? Prefer variety. |

## Cold Audience Hook Patterns

Every concept recommendation MUST pair with one of these hook patterns for the first 3 seconds. These are not separate concepts. They are applied as the opening beat of any concept.

### `pattern-interrupt`
**Pattern Interrupt.** Unexpected visual motion or scale change that breaks the scroll pattern. Color flash, full-screen text slam, reverse motion, glitch effect, or zoom burst.

- Best channels: Reels, TikTok, Instagram feed, LinkedIn feed
- Best pairings: animated-terminal, ui-animation, before-after, feature-checklist
- When to use: Product launches, tool demos, competitive comparisons, any content where the audience is not actively searching

### `curiosity-gap`
**Question / Curiosity.** Opens an information gap the viewer needs closed. Provocative statistic, contrarian claim, or "what if" framing. The opening text or number creates a need to keep watching.

- Best channels: LinkedIn feed, Twitter/X, website embed
- Best pairings: animated-infographic, metric-counter, timeline-journey, process-flow
- When to use: Thought leadership, industry reports, frameworks, methodology content, educational content

### `result-first`
**Result-First.** Shows the end result (the clean dashboard, the revenue number, the time saved) before explaining how. The viewer sees the outcome immediately and stays to learn the path.

- Best channels: All channels, especially paid ads
- Best pairings: metric-counter, before-after, social-proof-montage, animated-infographic
- When to use: Case studies, data-heavy assets, product updates with measurable impact, lead magnets with clear ROI

### `social-proof-hook`
**Social Proof Hook.** Opens with volume of evidence: customer count, logo wall flash, rating score, or a bold aggregate claim. Creates "everyone else knows this" pressure.

- Best channels: LinkedIn feed, website embed, sales decks
- Best pairings: social-proof-montage, metric-counter, feature-checklist
- When to use: Testimonial content, trust-building for established products, event recaps, any content with strong third-party credibility

### `loss-aversion`
**Warning / Loss Aversion.** Opens with the cost of inaction, a warning, or "stop doing X." Triggers the brain's loss aversion bias. Performance data shows these are among the highest-performing hooks for cold audiences because the threat of loss outweighs the promise of gain.

- Best channels: Paid ads (Meta, LinkedIn), Reels, TikTok
- Best pairings: before-after, pas-narrative, split-comparison, countdown-offer
- When to use: Competitive comparisons, pricing/offer content, churn prevention, any content where the status quo has a measurable cost

## Primary Concepts

Each concept below is a complete video archetype with a scene template. The agent selects from these based on the creative brief.

### `pas-narrative`: Problem-Agitation-Solution Visual Narrative

**Category:** Problem-Focused
**Description:** Three-act narrative that names the audience's pain, amplifies the emotional stakes, then presents the solution with proof. The most proven direct response structure adapted to motion graphics.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-4s | Bold text on brand background, subtle pulse/shake animation | Problem statement (1-2 lines) | Fade in |
| 2 | 4-8s | Pain amplification: icons, red/warning accents, visual tension, cascading failure visuals | Why the problem persists (1 line) | Cut or slide |
| 3 | 8-14s | Transition to solution: brand accent color takes over, clean layout | Solution introduction + key benefit | Color wipe |
| 4 | 14-18s | Feature/capability highlight with supporting visual | Second benefit or proof point | Slide |
| 5 | Last 3s | Brand card | Logo + tagline + URL + CTA | Fade |

**Best for:** Blog posts, playbooks, lead magnets, product updates, tool launches.
**Cold audience strength:** Strong. Problem statement in scene 1 creates immediate recognition without brand knowledge.
**Hook pattern affinity:** loss-aversion, curiosity-gap
**Animation style:** Spring (smooth) for entrances, ease-out for exits. Problem scenes use slightly faster timing to create tension. Solution scenes use slower, more confident pacing.
**Duration range:** 15-30s

### `before-after`: Before/After Transformation

**Category:** Problem-Focused
**Description:** Two states shown in contrast: the "without" state and the "with product" state. Can be implemented as a temporal transition (sequential reveal), spatial comparison (split-screen), wipe reveal, or snap transformation.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-3s | Full-screen "Before" label with problem visual (cluttered dashboard, manual process, old workflow) | "Before [Product/Asset]" | Fade in |
| 2 | 3-7s | Before state details: pain points highlighted with red/warning accents, icons for friction | 1-2 specific problems (e.g., "Manual reporting. 4 hours/week.") | Slide in elements |
| 3 | 7-8s | Split-screen wipe transition: brand accent color divider slides across | (transition moment) | Horizontal wipe |
| 4 | 8-13s | "After" state: clean, optimized, brand-colored visuals | Key result (e.g., "Automated. 12 minutes.") | Wipe reveal |
| 5 | 13-17s | After state details: metrics, improvements, green/success accents | 1-2 proof points | Slide in elements |
| 6 | Last 3s | Brand card | Logo + tagline + URL | Fade |

**Layout options:**
- Horizontal split: Before on left, After on right (1080x1080 and 1920x1080)
- Vertical split: Before on top, After on bottom (1080x1920)
- Sequential: Full-screen Before then wipe then full-screen After (all dimensions)
- Snap transformation: Instant switch with a visual trigger (button click, toggle)

**Best for:** Case studies, product updates, client testimonials, tool launches, playbooks, any asset with a clear transformation narrative.
**Cold audience strength:** Very strong. Before/after contrast is universally understood without context.
**Hook pattern affinity:** result-first, loss-aversion
**Animation style:** Before scenes use desaturated colors and heavier weight. After scenes use brand accent colors and lighter, more confident animation. The wipe transition should be decisive (300-500ms).
**Duration range:** 15-30s

### `day-in-life-contrast`: Day-in-the-Life Contrast

**Category:** Problem-Focused
**Description:** Shows the protagonist's day with vs. without the product. Not abstract pain points but concrete daily friction: the 47 browser tabs, the miscommunication in chat, the manual CSV export. Grounds the problem in lived experience.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-3s | Clock/calendar visual, "Your Tuesday morning" or similar | Time anchor + role identifier | Fade in |
| 2 | 3-7s | Cluttered workspace: tabs, notifications, manual spreadsheets, Slack messages | Specific friction point (e.g., "Exporting data. Again.") | Quick cuts |
| 3 | 7-9s | Frustration peak: visual chaos, desaturated | Time wasted (e.g., "2 hours before lunch.") | Cut |
| 4 | 9-13s | Screen clears. Clean interface. Product in action | "With [Product]: one click." | Dissolve |
| 5 | 13-17s | Calm workspace: focused, productive, time reclaimed | Result (e.g., "Done by 9:15 AM.") | Slide |
| 6 | Last 3s | Brand card | Logo + tagline + URL + CTA | Fade |

**Best for:** Product updates, product launches, blog posts about productivity, competitive comparisons.
**Cold audience strength:** Strong. Daily frustration is immediately relatable without brand knowledge.
**Hook pattern affinity:** loss-aversion, pattern-interrupt
**Animation style:** Before scenes: quick cuts, cluttered layouts, warm/tense colors. After scenes: smooth transitions, clean layouts, cool/confident colors.
**Duration range:** 15-30s

### `metric-counter`: Metric Counter Animation (Number Ticker)

**Category:** Data & Evidence
**Description:** Animated numbers counting from zero (or a baseline) to impressive figures. Revenue generated, time saved, users onboarded, errors prevented. The counting animation transforms static data into a story of progress. Can be the entire video or a focal element within another concept.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-4s | Large animated number counting up (e.g., 147%) on brand background | Metric label below the number | Scale up + fade |
| 2 | 4-8s | Second stat with different animation (scale, slide, typewriter) | Context line | Slide from right |
| 3 | 8-12s | Third stat or key quote in large type | Attribution or source | Fade + scale |
| 4 | 12-16s | Summary statement with all key numbers visible simultaneously | Synthesis line | Staggered fade-in |
| 5 | Last 3s | Brand card | Logo + tagline + URL | Fade |

**Best for:** Case studies, industry reports, event recaps, client testimonials, any asset with strong numbers.
**Cold audience strength:** Very strong. Numbers are universally scannable and create instant credibility.
**Hook pattern affinity:** result-first, social-proof-hook
**Animation style:** Mixed: each stat uses a different entrance animation for variety. Number counting uses linear interpolation. Text uses spring (snappy).
**Duration range:** 15-30s
**Remotion note:** AnimateNumber-style React component with spring interpolation. Frame-based system maps directly to incremental number reveals.

### `animated-infographic`: Animated Infographic / Data Visualization

**Category:** Data & Evidence
**Description:** Transforms charts, graphs, and statistics into dynamic visual stories. Animated bar charts, line graphs drawing in real time, pie charts assembling. Takes the Spotify Wrapped approach: data becomes narrative. 10x more effective than static infographics for retention.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-3s | Bold headline stat or provocative data point | "The state of [topic]" or hook number | Scale up |
| 2 | 3-8s | Animated bar chart or line graph drawing in real time | Axis labels + trend callout | Draw-in |
| 3 | 8-13s | Second visualization (different chart type for variety) | Key insight from the data | Morph or slide |
| 4 | 13-17s | Comparison overlay or annotated highlight on the data | "What this means" synthesis | Annotate |
| 5 | Last 3s | Brand card | Logo + tagline + URL + CTA | Fade |

**Best for:** Industry reports, data-heavy blog posts, research findings, benchmarking content, annual reviews.
**Cold audience strength:** Strong. Data visualizations are inherently attention-grabbing and scannable.
**Hook pattern affinity:** result-first, curiosity-gap
**Animation style:** Linear interpolation for chart drawing. Spring (smooth) for annotations and callouts. Staggered reveals for multi-element charts.
**Duration range:** 15-30s
**Remotion note:** D3.js integration for charts. Bar race charts are an existing Remotion template.

### `social-proof-montage`: Social Proof Montage

**Category:** Data & Evidence
**Description:** Cycles through testimonials, client quotes, logos, and metrics in a rhythm-driven sequence. Creates the impression of overwhelming evidence through volume and variety. Each "card" animates in, holds, and exits.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-3s | Hook stat or headline claim on brand background | Bold claim (e.g., "Trusted by 200+ B2B SaaS teams") | Fade in |
| 2 | 3-7s | First testimonial card: quote + name + company logo | Client quote (1-2 lines) | Slide in from right |
| 3 | 7-11s | Second testimonial card with different animation | Second client quote | Scale up + fade |
| 4 | 11-15s | Metrics bar: animated counters showing 2-3 aggregate results | Key numbers (e.g., "147% avg MQL increase") | Counter animation |
| 5 | 15-17s | Logo wall: 6-12 client/partner logos fade in on grid | "Join [X]+ companies" | Staggered grid fade |
| 6 | Last 3s | Brand card | Logo + tagline + URL + CTA | Fade |

**Sub-variants:**
- Logo wall animation: Client logos assembling into a grid
- Testimonial carousel: Quotes appearing, highlighting, cycling
- Review score cascade: G2/Capterra/star ratings animating in
- Social post wall: Real tweet/LinkedIn screenshots spotlighted

**Best for:** Client testimonials, case studies, event recaps, industry reports, any asset with third-party credibility.
**Cold audience strength:** Strong. Social proof is the fastest trust signal for cold audiences.
**Hook pattern affinity:** social-proof-hook, result-first
**Animation style:** Each testimonial card uses a different entrance for visual variety. Metric counters use linear interpolation. Logo grid uses staggered spring (smooth) entrances.
**Duration range:** 15-30s

### `kinetic-stomp`: Kinetic Typography - Stomp (Beat-Synced)

**Category:** Typography-Led
**Description:** Words placed and animated precisely synchronized to audio beats or visual rhythm. High-energy, bold, impactful. Each word or phrase lands with visual weight. Pure typographic motion: no UI, no screenshots, just words that hit.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-3s | Single bold word/phrase slams into center frame | Hook word (e.g., "FASTER") | Scale slam |
| 2 | 3-6s | Second phrase, different position, different size | Amplification (e.g., "THAN MANUAL WORKFLOWS") | Slide + rotate |
| 3 | 6-9s | Third phrase with color shift to brand accent | Key benefit or product name | Cut + color shift |
| 4 | 9-13s | Rapid sequence: 3-4 words/phrases appearing in rhythm | Feature list or value props | Staggered slam |
| 5 | 13-16s | All text clears. Single powerful statement | Summary claim | Dissolve to clean |
| 6 | Last 3s | Brand card | Logo + tagline + URL | Fade |

**Best for:** Product launches, event announcements, brand awareness, blog post promotion, any content that needs energy and urgency.
**Cold audience strength:** Very strong. Pure visual energy stops the scroll. No context needed.
**Hook pattern affinity:** pattern-interrupt, loss-aversion
**Animation style:** Scale slams (spring with high stiffness), rotation, position shifts. Mixed entrances per phrase. Frame-based sync to audio waveform if audio present.
**Duration range:** 10-20s
**Remotion note:** Frame-based sync to audio using Remotion's audio analysis. Each word/phrase is a separate React component with independent animation.

### `kinetic-scroll`: Kinetic Typography - Scrolling/Crawl

**Category:** Typography-Led
**Description:** Text moves along a plane (sideways, upward, Star Wars-style receding). Creates an elegant, smooth reading experience. Works for longer messages where the text itself IS the content. Calm authority, not energy.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-3s | Title/headline appears center frame | Hook statement | Fade in |
| 2 | 3-15s | Text begins scrolling upward or sideways. Key phrases highlighted in accent color as they pass center frame | Full message content with emphasis markers | Continuous scroll |
| 3 | 15-18s | Scroll slows and stops on final statement | Closing argument or CTA | Deceleration |
| 4 | Last 3s | Brand card | Logo + tagline + URL | Fade |

**Best for:** Thought leadership, manifestos, mission statements, event keynote summaries, long-form quote excerpts.
**Cold audience strength:** Moderate. Works best when the opening text line is provocative enough to compel reading.
**Hook pattern affinity:** curiosity-gap
**Animation style:** Linear scroll with variable speed (faster for context, slower for emphasis). Highlight uses spring (smooth) for color transition.
**Duration range:** 15-30s

### `caption-led-silent`: Caption-Led Silent Video

**Category:** Typography-Led
**Description:** Bold text overlays designed for sound-off viewing. 85% of social media users watch without sound. The text itself is the primary communication channel. Each frame delivers a complete thought. Background visuals support but do not carry the message.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-3s | Background visual (abstract, product, or contextual) | Hook statement in large bold type | Text slam |
| 2 | 3-7s | Subtle background shift or motion | Key insight line 1 | Fade swap |
| 3 | 7-11s | Background visual evolves | Key insight line 2 | Fade swap |
| 4 | 11-15s | Background intensifies or resolves | Proof point or data | Fade swap |
| 5 | 15-17s | Clean background | CTA statement | Fade swap |
| 6 | Last 3s | Brand card | Logo + tagline + URL | Fade |

**Best for:** Any social media content, blog post promotion, podcast episode promotion, event recaps, any content targeting mobile-first audiences.
**Cold audience strength:** Strong. Optimized for the default consumption mode (sound off, scrolling).
**Hook pattern affinity:** curiosity-gap, result-first
**Animation style:** Text emphasis animations (scale pulse, color highlight, underline draw). Background uses subtle parallax or slow motion. No audio dependency.
**Duration range:** 15-30s

### `screen-recording-overlay`: Screen Recording with Motion Graphics Overlay

**Category:** Product Demonstration
**Description:** Real product UI footage enhanced with animated callouts, zoom effects, cursor highlights, and explanatory graphics layered on top. Combines authenticity (real product) with clarity (animated annotations). Vector cursor replaces system cursor. Auto-zoom focuses on key interactions.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-3s | Product UI initial state, cursor enters | Hook: what the viewer will see | Fade in |
| 2 | 3-8s | Cursor navigates to feature. Animated zoom into interaction area | Step description + annotation callout | Zoom + annotate |
| 3 | 8-13s | Feature in action. Result appears. Callout highlights the key output | Benefit statement | Pan + callout |
| 4 | 13-17s | Second feature or capability demo with different annotation style | Second benefit | Zoom + annotate |
| 5 | Last 3s | Brand card | Logo + tagline + URL + CTA | Fade |

**Best for:** Product updates, product launches (non-CLI), SaaS feature demos, UI/UX improvements, onboarding content.
**Cold audience strength:** Moderate. Requires the UI to be visually interesting. Best paired with a strong text hook in scene 1.
**Hook pattern affinity:** pattern-interrupt, result-first
**Animation style:** Smooth zoom (ease-in-out), spring (snappy) for callout annotations, linear for cursor movement.
**Duration range:** 15-30s
**Remotion note:** Video file composited with SVG overlays, animated zoom crops using scale() and translate(), timed annotation components.

### `ui-animation`: UI/UX Animation (Interface Mockup)

**Category:** Product Demonstration
**Description:** Product interface recreated as animated components rather than screen-recorded. Allows perfect timing, idealized data, and branded visual treatment. The "product in motion" without the messiness of real screen recordings.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-3s | Simplified UI shell appears with branded styling | Hook: product name or category | Spring entrance |
| 2 | 3-8s | Data populates into UI (numbers counting, rows appearing, charts drawing) | Feature description | Staggered fill |
| 3 | 8-13s | User interaction simulated: click, toggle, filter, or drag | Benefit of the interaction | Animate interaction |
| 4 | 13-17s | Result state: clean output, success state, or transformed view | Proof point or result | Morph transition |
| 5 | Last 3s | Brand card | Logo + tagline + URL + CTA | Fade |

**Best for:** Product launches (non-CLI), SaaS product marketing, product update announcements, feature comparison pages.
**Cold audience strength:** Strong when the UI communicates value visually (dashboards, analytics, visual builders). Weaker for abstract/text-heavy UIs.
**Hook pattern affinity:** pattern-interrupt, result-first
**Animation style:** Spring entrances for UI elements. Linear for data population. Ease-in-out for interactions.
**Duration range:** 15-30s
**Remotion note:** Native strength. React components ARE UI components. Building animated mockups in Remotion is building interfaces with animated state transitions.

### `timeline-journey`: Timeline / Journey Animation

**Category:** Narrative & Storytelling
**Description:** Visualizes a progression through time or stages. A continuous line, road, or path connects waypoints. Each waypoint appears as the path reaches it. Works for customer journeys, product evolution, historical timelines, or multi-step transformations.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-3s | Starting waypoint appears with context label | "Where [subject] started" or time anchor | Fade in + path starts |
| 2 | 3-7s | Path draws to second waypoint. Waypoint springs in | Stage 2 description | Path draw + spring |
| 3 | 7-11s | Path draws to third waypoint | Stage 3 description | Path draw + spring |
| 4 | 11-15s | Path reaches final waypoint. Emphasis animation | Destination/result | Path draw + pulse |
| 5 | 15-17s | Full path visible. Zoomed out view | Summary statement | Zoom out |
| 6 | Last 3s | Brand card | Logo + tagline + URL | Fade |

**Best for:** Customer journeys, product evolution stories, multi-step processes, historical timelines, event recaps, career progression content.
**Cold audience strength:** Moderate. The journey metaphor is universally understood, but requires 5-7 seconds to build meaning.
**Hook pattern affinity:** curiosity-gap, result-first (show destination first, then rewind)
**Animation style:** SVG path drawing (stroke-dasharray/dashoffset). Spring entrances for waypoints. Linear interpolation for path progression.
**Duration range:** 15-30s

### `process-flow`: Process Flow Animation

**Category:** Narrative & Storytelling
**Description:** Progressive build of a conceptual framework, process diagram, or methodology. Each scene adds a node/step to the visual. The full picture assembles before the viewer's eyes. Use cases ground the abstract framework in concrete applications.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|
| 1 | 0-3.5s | Bold text on brand background, staggered line reveals | Hook: name the tension | Staggered fade + Y-translate |
| 2 | 3.5-6s | Logo/wordmark spring entrance | Brand identification + tagline | Spring scale |
| 3 | 6-8s | Tagline (uppercase, small, accent) + headline (large) | Framework title | Fade + spring |
| 4-7 | 8-19s | Progressive node build: each node springs in with connector paths drawing between them | Node name + one-line description per stage | Blur-to-sharp + spring |
| 8 | 19-22s | All nodes visible, loop/flow animation active | Compounding/reinforcement message | Acceleration + glow |
| 9 | 22-28s | Framework dims. Use cases cycle (fade in/hold/fade out, ~1.5s each) | 3-5 concrete applications | Crossfade |
| 10 | Last 2-3s | Brand card | URL + closing tagline + CTA | Spring |

**Flow types:**
- Linear flow: Step 1 → Step 2 → Step 3
- Branching flow: Decision points split into paths
- Circular/loop flow: Continuous recurring process
- Layered flow: Multiple parallel processes converging
- Hub-and-spoke: Central concept with radiating connections

**Best for:** Proprietary frameworks, methodologies, process explanations, "how it works" content, strategy documents, approach overviews.
**Cold audience strength:** Moderate-strong. The progressive build is inherently engaging, but requires setup time. Front-load with a tension hook.
**Hook pattern affinity:** curiosity-gap, loss-aversion
**Animation style:** Spring entrances for nodes. SVG stroke-dashoffset path drawing for connectors. Pulse/comet animation for loops. V2 polish: blur-to-sharp materializations, glow trails, comet tail pulse, accent-bar descriptions, colored badges.
**Duration range:** 20-30s

### `feature-checklist`: Feature Checklist Animation

**Category:** Comparison & Competitive
**Description:** Animated table or list where features check off sequentially. Can be a single-product capability showcase or a competitive comparison where your product shows checkmarks and competitors show X marks. The classic comparison page brought to motion.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|

**Best for:** Competitive comparisons, alternative pages, pricing tier breakdowns, feature announcements, product capability overviews.
**Cold audience strength:** Strong. Checklists are scannable and create clear differentiation instantly.
**Hook pattern affinity:** loss-aversion, result-first
**Animation style:** Staggered spring entrances per row. Check marks use spring (bouncy). X marks use linear (flat, no bounce). Highlight row uses accent color pulse.
**Duration range:** 15-30s

**Category:** Comparison & Competitive
**Description:** Simultaneous display of two approaches, products, or states. Left panel shows competitor/old way, right panel shows your product/new way. Both sides animate in parallel for real-time comparison.

**Hook pattern affinity:** loss-aversion, pattern-interrupt

### `countdown-offer`: Countdown / Price Reveal / Offer Animation

**Category:** Urgency & Conversion
**Description:** Builds to a price point, offer, or deadline through visual buildup. Value stacking before reveal, crossed-out original prices, progressive discount animation, or countdown timers creating urgency. Research shows 20% uplift in CTA click-through rates when countdown timers are present.

**Scene Template:**

| # | Time | Visual | Overlay Text | Transition |
|---|------|--------|-------------|------------|

| 2 | 3-6s | Value items stack up (2, 3, 4) | Additional inclusions | Staggered stack |
| 3 | 6-9s | Total value displayed, then crossed out | "Total value: $X" (strikethrough) | Draw strikethrough |
| 4 | 9-13s | Offer price reveals with emphasis | "Today: $Y" (accent color, larger) | Scale up + glow |

**Hook pattern affinity:** loss-aversion, result-first
**Animation style:** Staggered entrances for value stack. Strikethrough uses linear draw. Price reveal uses spring (bouncy) with glow. Countdown uses linear tick.
**Duration range:** 15-30s

## Visual Style Modifiers

These are not standalone concepts. They are visual treatments applied to any primary concept to enhance the aesthetic. Specify one modifier per concept when it adds value.

### `abstract-shapes`
**Abstract Shape / Mood Animation.** Bold compositions, flowing shapes, dynamic color transitions, non-literal visuals. Used for brand emotion, mood-setting, and innovative positioning. Apply as background motion behind any primary concept.

### `liquid-morph`
**Liquid Motion / Morphing Transition.** Simulates flowing movement with organic animations. Shapes morph from simple to detailed, representing transformation or growth. Apply to transitions between scenes or as element entrances.
**Remotion note:** SVG path morphing (flubber library) or CSS clip-path animations.

### `parallax-depth`
**Parallax Depth Layer Animation.** Multiple layers moving at different speeds to create depth. Foreground elements move faster than background. Adds sophistication to any flat 2D concept.
**Remotion note:** Multiple div layers with different interpolation speeds on translateY/translateX.

### `whiteboard-reveal`
**Whiteboard / Hand-Drawn Reveal.** Hand-drawn visuals appearing in real time with a sketch-based style. Step-by-step reveal creates the feeling of "watching someone think through a problem." Signals authenticity.
**Remotion note:** SVG path animation with stroke-dasharray/stroke-dashoffset.

### `mixed-media`
**Mixed Media Collage.** Combines different visual elements: photos, illustrations, text, textures, gradients. Scrapbook aesthetic with intentional layering. Signals creativity and human craft.
**Remotion note:** Layering texture images with CSS mix-blend-mode and compositing cut-out style assets.

### `audio-reactive`
**Audio-Reactive Visualization.** Visual elements responding to soundtrack or voiceover. Waveforms, pulsing shapes, rhythm-driven motion.
**Remotion note:** Remotion's `useAudioData()` and `visualizeAudio()` APIs.

### `isometric`
**Isometric Environment Animation.** 2.5D viewpoint simulating three dimensions on a 2D plane. All axes at 120-degree angles. Works for visualizing workflows, architectures, or multi-layer systems.
**Remotion note:** SVG transformations and CSS perspective. Isometric grids are composable as React components.

### `deep-glow-neon`
**Deep Glow / Neon Tech Aesthetic.** Intense light blooms, layered neons, radiant gradients. Feels immersive and futuristic. Strong for developer tools and technical products.
**Remotion note:** CSS box-shadow, filter: blur(), and animated gradient backgrounds.
